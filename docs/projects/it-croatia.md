I started this project in the early stages of my academic journey while exploring communities, conferences, and first job opportunities to start my career. As I entered the world of the IT industry, I decided to create a platform that could serve as a comprehensive resource for others starting (or continuing) their career paths.

The process of collecting data involves a combination of automatic and manual data extraction and filtering of publicly available sources. Automatic data collection is performed using a combination of custom web scraping tools and Python scripts, while manual data collection is performed by searching for information in the public domain.

The initial launch of the project was announced on [Reddit](https://www.reddit.com/r/CroIT/comments/1bfba43/kolekcija_it_kompanija_konferencija_edukacija/). I also shared it on LinkedIn, Facebook, Twitter, and Bluesky to gain user traction and gather feedback from the community. As I had no prior experience with releasing public projects, the initial response was overwhelming but invaluable. The inputs I received helped shape the platform in many ways.

While observing the heatmaps data ‚Äî based on 200,000+ recorded interactions ‚Äî the analysis reveals interesting patterns in user behavior.

Starting from the top of the page, nearly all users clicked the top-right notifications icon (üîî). I used this opportunity to explore how users instinctively react to notification-like elements, even when repurposed. In this case, the notification icon serves as a disguise for the changelog, drawing immediate attention to the latest updates and useful tips.

Desktop users frequently clicked on the search input field as their second action, indicating a clear intent to quickly locate specific information.

In contrast, mobile users interacted more with the table data filters to narrow down options efficiently, as scrolling through long lists on a smaller screen can be tedious. This behavior highlights the importance of providing intuitive filtering options for mobile users, who navigate differently due to touch-based interaction and limited screen space.

I tried a few tools: Vercel, Cloudflare, Google Analytics, Microsoft Clarity, and PostHog. At some point, I had all of them running, so I used a piece of paper to track the data from each.
